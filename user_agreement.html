Пользовательское соглашение

Настоящее Пользовательское соглашение устанавливает порядок использования веб‑приложения Queue. Используя Приложение, вы подтверждаете, что ознакомились с этим Соглашением и согласны с его условиями.

1. Предмет соглашения

1.1. Queue предоставляет функционал для электронной записи в очередь на сдачу лабораторных работ. Сервис обеспечивает автоматическое получение расписания через API университета, а также управление очередями через веб‑интерфейс.

2. Основные термины

  1. Пользователь — физическое лицо, прошедшее регистрацию в Приложении.
  2. Организатор — администратор Приложения или ответственные лица университета.
  3. Очередь — упорядоченный список записавшихся пользователей на сдачу лабораторной работы по предмету.

3. Доступ и регистрация

3.1. Для работы с Приложением требуется регистрация: имя, e‑mail, номер группы, подгруппа и пароль. После регистрации пользователь обязан подтвердить e‑mail.
3.2. Доступ осуществляется по e‑mail и паролю. Пользователь обязан обеспечить конфиденциальность своих учетных данных и незамедлительно уведомлять службу поддержки о факте несанкционированного доступа.

4. Функции сервиса

4.1. Получение расписания: Приложение автоматически запрашивает и получает данные расписания конкретной учебной группы через API университета (при наличии доступа/токена).
4.2. Управление очередью: пользователь может выбрать предмет и лабораторную работу, занять место в очереди, просмотреть свою позицию, покинуть очередь, отправить или отменить запрос на обмен местами с другим студентом.
4.3. Уведомления: при согласии/отказе по обмену местами, а также при ключевых событиях система отправляет электронные уведомления на e‑mail участников.

5. Ограничения и условия использования

5.1. Студент может занимать очередь только для своей учебной группы и только за себя; множественные записи на разные предметы и лаборатории возможны в пределах расписания.
5.2. Для формирования новой очереди и корректного отображения свободных слотов необходимо, чтобы API университета был доступен; при недоступности API функционал может быть ограничен.
5.3. Приложение предназначено для использования на настольных браузерах и требует доступа в интернет.

6. Сохранение и обработка данных

6.1. Приложение хранит в базе данных (СУБД) информацию об очередях, истории сдач, учётных записях и логах операций. Персональные данные (имя, e‑mail, группа) обрабатываются в целях предоставления сервиса и в соответствии с политикой конфиденциальности.

7.3. Пользователь отвечает за сохранность своих учётных данных и обязан не передавать их третьим лицам.
